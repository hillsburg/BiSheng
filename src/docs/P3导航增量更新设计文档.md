# P3 导航增量更新设计文档

## 一、概述

P3 在 P1 `INavigationStore` 之上，将同步完成后的 **全量 `Refresh()`** 替换为 **按批增量 patch**，在常见小批量变更下避免重建整棵 `FolderTree` / `NoteList`。

| 子项 | 目标 | 状态 |
|------|------|------|
| **P3-1** | `SyncNavigationDelta` + `SyncService.OnSyncComplete` 上报 | ✅ |
| **P3-2** | `NoteListViewModel.ApplyNavigationDelta`（并列模式） | ✅ |
| **P3-3** | `FolderTreeViewModel.ApplyNavigationDelta`（Folder 树） | ✅ |
| **P3-4** | `NavigationRefreshPolicy` fallback（搜索/重命名/大批量/move） | ✅ |
| **P3-5** | 归纳模式树内 Note + 收藏区 `RebuildFavorites` | ✅ |
| **P3-6** | `NavigationRefreshPolicy` / `NoteList` 增量单测 | ✅ |

---

## 二、数据流

```
SyncService（Pull / SignalR / Push 完成）
  → OnSyncComplete(SyncNavigationDelta)
  → NavigationStore.RefreshAfterSync(delta, isTreeMode)
       ├─ 编辑中 / 内联重命名 → 2s 延迟 + 合并 pending delta
       ├─ NavigationPatcher.TryApplyIncremental
       │     ├─ true  → FolderTree/NoteList 局部 patch + Editor.TryReloadIfNoteAffected
       │     └─ false → fallback 全量 Refresh（P1 行为）
       └─ Empty delta → 无 UI 重建（仅编辑器按需 reload）
```

---

## 三、核心类型

| 类型 | 路径 | 职责 |
|------|------|------|
| `SyncNavigationDelta` | `Services/Navigation/SyncNavigationDelta.cs` | 同步批次描述：`Empty` / `FullRefresh` / `Changes` |
| `NavigationChange` | 同上 | 单条 Note/Folder 导航影响 |
| `NavigationDeltaBuilder` | `Services/Navigation/NavigationDeltaBuilder.cs` | Pull **Apply 前**从 `ChangeDto` 构建（检测 `ParentFolderChanged`） |
| `NavigationRefreshPolicy` | `Services/Navigation/NavigationRefreshPolicy.cs` | fallback 条件 |
| `NavigationPatcher` | `Services/Navigation/NavigationPatcher.cs` | 编排 VM 增量应用 |
| `LocalFolderMerger` | `Services/Navigation/LocalFolderMerger.cs` | Folder DB → 树节点合并 |

---

## 四、Fallback 条件（全量 Refresh）

满足任一即 `NavigationPatcher` 返回 false：

| 条件 | 原因 |
|------|------|
| `delta.RequiresFullRefresh` | 全量同步、冲突解决覆盖远端等 |
| 搜索框非空 | 过滤结果无法局部推断 |
| 内联重命名进行中 | 避免打断 IME / 光标 |
| `ParentFolderChanged` | 树结构需重挂 |
| 变更数 > 50 | 批量成本低于 patch 复杂度 |

Note **跨文件夹移动**在树模式（`IncludeNotes`）下 fallback；并列模式由 `NoteList` 从当前列表移除。

## 五、SyncService 上报规则

| 场景 | Delta |
|------|-------|
| Pull 无变更 | `Empty`（不触发 UI 刷新） |
| Pull 有变更 | `FromChanges(NavigationDeltaBuilder…)` |
| Pull `RequiresFullSync` | `FullRefresh` |
| SignalR 通知 → 防抖 Pull | 同 Pull（通知无 Payload，不就地应用） |
| SignalR 补偿 Pull | 同 Pull |
| Push 成功无新 Pull | `Empty` |
| Push 触发全量重建 | `FullRefresh` |
| 冲突 Keep Remote | `FullRefresh` |
| 冲突 Keep Local / Merged | 由后续 `PushAndPull` 上报 |

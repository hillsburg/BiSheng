# P4 读模型与展示层隔离设计文档

## 一、目标

P3 减少了全量 `Refresh`，但 Sync 仍通过 `OnSyncComplete` 事件直接驱动 UI。P4 建立 **数据层写入 → 读模型发布 → 展示协调** 三层边界：

| 层 | 职责 | 不应知道 |
|----|------|----------|
| **SyncService** | Push/Pull、写 SQLite、`_syncLock` | FolderTree / Editor / Dispatcher |
| **INavigationReadModel** | 发布 `SyncNavigationDelta` | 如何 patch UI |
| **NavigationPresentationCoordinator** | Dispatcher、deferred、布局模式 | HTTP / SignalR |
| **NavigationStore** | 导航 VM patch / fallback Refresh | Sync 何时完成 |
| **IEditorSessionService** | 版本感知编辑器合并 | Pull 协议 |

---

## 二、数据流（P4）

```
SyncService（写 SQLite 后）
  → INavigationReadModel.Publish(delta)
       ↓ Changed
NavigationPresentationCoordinator（UI 线程）
  ├─ 编辑中 / 重命名 → 2s deferred + merge delta
  ├─ NavigationStore.ApplyRemoteDelta(delta, layoutMode)
  └─ EditorSessionService.ApplyRemoteChanges(delta)
         └─ 仅当 openNote.Version > loadedVersion 或 FullRefresh
```

**MainViewModel 不再订阅 Sync 完成事件**；仅通过 `INavigationLayoutMode` 同步 `IsTreeMode`。

## 四、编辑器会话层

- `NotifyNoteOpened(noteId, version)`：在 `NavigationStore.NoteSwitching` 时调用
- `ApplyRemoteChanges`：读模型变更后，**版本未前进则跳过** `TryReloadCurrentNoteFromDb`
- `IsEditingSessionActive` 仍由 `EditorViewModel` 定义；推迟逻辑集中在 **Coordinator**，不再分散在 `NavigationStore`

---

## 五、与 P3 关系

| P3 | P4 |
|----|-----|
| `SyncNavigationDelta` | 保留，作为读模型 payload |
| `NavigationPatcher` | 保留；移除 Editor 参数 |
| `RefreshAfterSync` | 重命名为 `ApplyRemoteDelta`，仅由 Coordinator 调用 |
| `SyncService.OnSyncComplete` | **删除** → `INavigationReadModel.Publish` |

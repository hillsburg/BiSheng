# 同步协议契约共享库设计文档

## 背景

BiSheng 的客户端（`BiSheng.Latte`，WPF）与服务端（`BiSheng.Server`，ASP.NET Core）通过 HTTP/JSON + SignalR 进行增量同步。同步协议中，`EntityType`（实体类型）和 `Action`（操作类型）以字符串形式在两端之间传输：

```
客户端 Push → {"entityType": "Note", "action": "Create", ...} → 服务端
服务端 Pull → {"entityType": "Folder", "action": "Update", ...} → 客户端
```

**这两组字符串是同步协议的契约，两端必须完全一致。** 若不一致，同步会静默失败——变更被跳过、数据丢失，且无任何报错。

## 问题

最初 `EntityTypes`（`"Folder"` / `"Note"`）和 `ChangeActions`（`"Create"` / `"Update"` / `"Delete"`）以 magic string 形式散布在客户端和服务端各自的代码中，存在两类风险：

1. **拼写错误**：某处写成 `"Create"` 另一处写成 `"Ceate"`，编译通过但运行时失败
2. **分别维护**：两端各有一份定义，新增类型时若只改一边，协议断裂

## 方案：`BiSheng.Shared` 共享项目

### 项目结构

```
src/BiSheng.Shared/
├── BiSheng.Shared.csproj   ← netstandard2.0，零依赖
├── EntityTypes.cs           ← 实体类型常量（Folder、Note）
├── ChangeActions.cs         ← 操作类型常量（Create、Update、Delete）
└── Sync/SyncDtos.cs         ← Pull/Push DTO（含 NextSince / HasMore / RequiresFullSync）
```

### 引用关系

```
BiSheng.Latte (net8.0-windows)
    └── ProjectReference → BiSheng.Shared

BiSheng.Server (net8.0)
    └── ProjectReference → BiSheng.Shared
```

### 目标框架选择：`netstandard2.0`

| 考量 | 说明 |
|---|---|
| 库内容 | 仅含 `const string`，无任何框架依赖 |
| 兼容性 | `.NET Framework 4.6.1+` / `.NET Core 2.0+` / `.NET 5+` 全平台支持 |
| 扩展性 | 未来移动端、MAUI 等新客户端可直接引用，无需重建 |

## 常量定义

### `EntityTypes` — 实体类型

```csharp
public static class EntityTypes
{
    public const string Folder = nameof(Folder);  // "Folder"
    public const string Note   = nameof(Note);    // "Note"
}
```

### `ChangeActions` — 操作类型

```csharp
public static class ChangeActions
{
    public const string Create = nameof(Create);  // "Create"
    public const string Update = nameof(Update);  // "Update"
    public const string Delete = nameof(Delete);  // "Delete"
}
```

使用 `nameof()` 确保常量名与值始终一致，重构时重命名不会破坏字符串值。

## 扩展指南

新增实体类型或操作类型时，**只需修改本项目的对应文件**：

```csharp
// EntityTypes.cs — 新增 Tag 类型
public const string Tag = nameof(Tag);

// ChangeActions.cs — 新增 Move 操作
public const string Move = nameof(Move);
```

客户端和服务端通过 `ProjectReference` 自动获取新常量，编译时即可发现未处理的 `switch` / `if` 分支。

## 使用规范

- **禁止**在代码中使用 `"Folder"`、`"Note"`、`"Create"`、`"Update"`、`"Delete"` 等 magic string
- **必须**使用 `EntityTypes.Folder`、`ChangeActions.Create` 等常量引用
- 涉及 `EntityType` 或 `Action` 字段的所有文件需添加 `using BiSheng.Shared;`

# 主题配色系统设计文档

## 一、架构概述

本项目采用 **统一主题定义 + 核心色/辅助色分离** 的配色架构：

```
┌─────────────────────────────────────────────┐
│  ThemeDefinition (31 个颜色属性)              │
│  ├── 全局 UI 色 (19 tokens)                  │
│  │   ├── 核心色 (14) ← 用户在 UI 中编辑      │
│  │   └── 辅助色 (5)  ← DeriveAuxiliaryColors │
│  └── 编辑器色 (14 tokens)                    │
│      └── 全部在 UI 中暴露                     │
└──────────────────┬──────────────────────────┘
                   │
        ApplyToResources()
                   ↓
        App.xaml ResourceDictionary
        (DynamicResource 自动刷新)
```

## 三、辅助色自动推导

`DeriveAuxiliaryColors()` 在 `ApplyToResources()` 时执行，从核心色计算辅助色：

| 辅助色 | 推导公式 |
|--------|--------|
| `BorderLight` | Border 向 BgCanvas 淡出（55%） |
| `AccentHover` | Accent 加深 15% |
| `SelectedBorder` | Accent 提亮 10% |
| `ToolbarHover` | ToolbarBg 提亮 12% |
| `ToolbarIcon` | ToolbarText 提亮 8% |

> **注意**：`SurfaceAlt` 和 `TextMuted` 不再自动推导，由用户在 UI 中直接控制（状态栏配色）。

### 颜色工具方法

```csharp
BlendHex(a, b, factor)   // 混合两色，factor 为 a 的权重
DarkenHex(hex, amount)   // 加深（乘以 1-amount）
LightenHex(hex, amount)  // 提亮（向白色插值）
GetLuminance(hex)        // 计算相对亮度（0~1）
```

## 四、内置预设

| 预设 | 风格 | 背景亮度 |
|------|------|---------|
| Light | 清新白底，标准配色 | 高 |
| Dark | 护眼暗色，减少眩光 | 低 |
| Latte | 暖咖啡色调，温暖优雅 | 中高 |

内置预设为只读，用户可通过"复制并自定义"创建可编辑副本。

---

## 五、持久化

- 主题以 JSON 文件存储在用户目录
- 文件名 = `{主题名}.json`
- 包含全部 31 个颜色属性（含辅助色），确保加载时无需重新推导

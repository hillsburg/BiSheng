# P5 单向数据流与过滤投影设计文档

## 一、目标

P4 将 Sync 与 UI 拆为读模型边界，但本地 CRUD 与搜索仍直接调用 `Refresh()`。P5 统一为：

| 来源 | 路径 |
|------|------|
| 远端 Sync | `SyncService` → `INavigationReadModel.Publish(DataChange)` |
| 本地 CRUD | `INavigationMutationPublisher` → 读模型 |
| 搜索过滤 | `NavigationFilterBridge` → `FilterOnly` 投影 |
| 布局切换 | `NotifyLayoutRebuild()` → `LayoutRebuild` 投影 |

---

## 二、投影类型

```csharp
enum NavigationProjectionKind
{
    DataChange,    // 增量 patch 或 fallback 全量
    FilterOnly,    // 仅重算搜索过滤，不碰编辑器
    LayoutRebuild  // 模式/布局/回收站恢复等全量导航重建
}
```

`NavigationProjectionUpdate` 包装 `Kind` + `SyncNavigationDelta`。

---

## 三、核心类型

| 类型 | 职责 |
|------|------|
| `INavigationMutationPublisher` | FolderTree / NoteList 本地变更唯一出口 |
| `NavigationMutationPublisher` | 构造 `NavigationChange` 并同步发布（`waitForPresentation: true`） |
| `NavigationFilterBridge` | `SearchChanged` → `NotifyFilterChanged()` |
| `NavigationPresentationCoordinator` | 按 `Kind` 分发到 Store / EditorSession |

---

## 四、展示层分发

```
Coordinator.HandleProjectionUpdate
  FilterOnly     → NavigationStore.ApplyFilterProjection
  LayoutRebuild  → NavigationStore.ApplyLayoutRebuild
  DataChange     → deferred? → ApplyRemoteDelta + EditorSession（同 P4）
```

本地 CRUD 与远端 Sync **共用** `ApplyRemoteDelta` / `NavigationPatcher` 增量路径。

---

## 五、移除的直接 Refresh 路径

- `MainViewModel`：`SearchChanged`、`OnFlagsChanged`、同步完成后的树刷新
- `FolderTreeViewModel` / `NoteListViewModel`：CRUD 后的 `Refresh()`
- `NavigationContextMenus`：删除笔记后的 `FolderTree.Refresh()`
- 布局/回收站/冲突对话框：改为 `NotifyLayoutRebuild()`

`Refresh()` 仍保留为 Store / VM 内部实现，**不再由业务 scattered 调用**（除 Store 自身 fallback）。

---

## 六、与 P4 关系

| P4 | P5 |
|----|-----|
| `Publish(SyncNavigationDelta)` | `Publish(NavigationProjectionUpdate)` |
| 仅 Sync 写入 | Sync + 本地 CRUD + 过滤 + 布局 |
| Coordinator 只处理 DataChange | Coordinator 三分投影 |

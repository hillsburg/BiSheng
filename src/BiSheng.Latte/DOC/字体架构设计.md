# 字体架构设计

## 概述

BiSheng Latte 采用**分层字体架构**，按使用场景严格分离字体职责，避免单一全局字体导致的 UI 可读性问题和图标字体冲突。

**正文字体不随应用分发**：编辑器字体仅使用用户系统已安装的字体，避免字体版权风险；用户可在「外观设置」中按主题自选。

## 字体资源定义

在 `App.xaml` 中定义四种字体资源：

| 资源 Key | 字体 | 使用场景 |
|----------|------|----------|
| `UIFont` | Microsoft YaHei / Segoe UI | UI 元素（按钮、菜单、设置窗口、树、列表） |
| `ContentFont` | Microsoft YaHei / Segoe UI（默认） | 编辑器正文；启动后由主题与用户设置覆盖 |
| `CodeFont` | Consolas / Fira Code | 代码块（预留） |
| `IconFont` | iconfont（iconfont.ttf） | 工具栏图标按钮（应用自带图标字体） |

### App.xaml 配置

```xml
<!-- UI 字体：清晰可读的系统字体 -->
<FontFamily x:Key="UIFont">Microsoft YaHei, Segoe UI</FontFamily>

<!-- 内容字体：默认微软雅黑；运行时由主题/设置覆盖 -->
<FontFamily x:Key="ContentFont">Microsoft YaHei, Segoe UI</FontFamily>

<!-- 代码字体：等宽字体 -->
<FontFamily x:Key="CodeFont">Consolas, Fira Code, Courier New</FontFamily>

<!-- 图标字体（应用资源，非正文字体） -->
<FontFamily x:Key="IconFont">pack://application:,,,/Resources/Fonts/iconfont.ttf#iconfont</FontFamily>
```

## 按主题的编辑器字体

- 每个主题在 `ThemeDefinition.ContentFontFamily` 有默认字体栈（均为 `Microsoft YaHei, Segoe UI`）。
- 用户在外观设置中为某主题选择的字体保存在 `AppearanceSettings.ThemeContentFonts`。
- `FontCatalog` 仅解析系统已装字体，并自动追加 `Microsoft YaHei, Segoe UI` 作为最终回退。

## 全局样式绑定

所有 UI 控件的全局样式统一使用 `UIFont`：

```xml
<Style TargetType="Window">
    <Setter Property="FontFamily" Value="{StaticResource UIFont}"/>
</Style>

<Style TargetType="TextBlock">
    <Setter Property="FontFamily" Value="{StaticResource UIFont}"/>
</Style>

<Style TargetType="Button">
    <Setter Property="FontFamily" Value="{StaticResource UIFont}"/>
</Style>
```

## 编辑器字体传递

BiSheng.Editor 作为独立程序集，通过 **DependencyProperty** 接收宿主字体配置，实现松耦合。

### 传递流程

```
App.xaml (ContentFont 定义)
    ↓ DynamicResource 引用
MainWindow.xaml (EditorFontFamily="{DynamicResource ContentFont}")
    ↓ DependencyProperty 绑定
MarkdownEditorControl.xaml.cs (EditorFontFamilyProperty)
    ↓ OnEditorStyleChanged 回调
ApplyEditorStyle()
    ↓ 赋值
AvalonEdit Editor.FontFamily + _theme.BaseFontFamily
```

### 关键代码

**MainWindow.xaml：**
```xml
<md:MarkdownEditorControl
    EditorFontFamily="{DynamicResource ContentFont}"
    EditorFontSize="15"/>
```

**MarkdownEditorControl.xaml.cs：**
```csharp
public static readonly DependencyProperty EditorFontFamilyProperty =
    DependencyProperty.Register(nameof(EditorFontFamily), typeof(FontFamily),
        typeof(MarkdownEditorControl),
        new PropertyMetadata(new FontFamily("Segoe UI, Microsoft YaHei"), OnEditorStyleChanged));

private void ApplyEditorStyle()
{
    Editor.FontFamily = EditorFontFamily;           // 赋值给内部 AvalonEdit
    _theme.BaseFontFamily = EditorFontFamily;       // 同步给主题（语法渲染用）
}
```

## 图标字体使用

### 引用方式

```xml
<TextBlock Text="&#xeb6d;"
           FontFamily="{StaticResource IconFont}"
           FontSize="16"/>
```

### 图标资源字典

`Resources/Fonts/IconResources.xaml` 提供命名字符串资源。

使用方式：
```xml
<TextBlock Text="{StaticResource Icon.Search}"
           FontFamily="{StaticResource IconFont}"/>
```

### ControlTemplate 中的字体传递

当按钮使用自定义 `ControlTemplate` 时，`ContentPresenter` 需显式绑定字体属性：

```xml
<ContentPresenter
    HorizontalAlignment="Center"
    VerticalAlignment="Center"
    TextBlock.FontFamily="{TemplateBinding FontFamily}"
    TextBlock.FontSize="{TemplateBinding FontSize}"
    TextBlock.Foreground="{TemplateBinding Foreground}"/>
```

## 设计决策

### 为什么不内置正文字体？

| 考量 | 说明 |
|------|------|
| 版权合规 | 正文字体授权复杂，随应用分发风险高 |
| 用户习惯 | 用户可自行安装霞鹜文楷、思源宋体等并在设置中选择 |
| 包体积 | 不嵌入大型 ttf，安装包更小 |

### 为什么 UI 与正文分层？

| 问题 | 说明 |
|------|------|
| 小字号可读性 | 装饰性字体在 12–14px UI 下可读性差 |
| 系统一致性 | Windows 用户习惯微软雅黑/Segoe UI 的 UI 渲染 |
| 图标字体冲突 | 全局字体样式会干扰 iconfont 等专用字体的渲染 |
| 功能适配 | 代码块需要等宽字体 |

### 字体资源 Key 命名规范

- 所有字体 Key 使用 **PascalCase**（如 `UIFont`、`ContentFont`、`IconFont`）
- 引用时大小写严格匹配，避免 `XamlParseException`
- 已废弃的 `DefaultFont` 键名禁止使用

## 文件结构

```
BiSheng.Latte/
├── Resources/
│   └── Fonts/
│       ├── iconfont.ttf               # 图标字体（应用资源）
│       └── IconResources.xaml         # 图标字符串资源字典
├── App.xaml                            # 字体资源定义 + 全局样式
└── MainWindow.xaml                     # 编辑器字体绑定
```

## 常见问题排查

### 1. 自定义字体（iconfont）不生效

**原因：** SDK 项目不会自动将 `.ttf` 文件嵌入为 Resource。

**解决：** 必须在 `.csproj` 中显式声明：
```xml
<ItemGroup>
    <Resource Include="Resources\Fonts\iconfont.ttf" />
</ItemGroup>
```

### 2. 编辑器字体显示为雅黑而非所选字体

**原因：** 所选字体未在系统中安装，WPF 按回退链使用 `Microsoft YaHei`。

**解决：** 在系统中安装目标字体后，在外观设置中重新选择。

### 3. ControlTemplate 中字符串内容字体不生效

**原因：** `ContentPresenter` 上的 `TextBlock.FontFamily` 附加属性 + `TemplateBinding` 对字符串内容不起作用。

**解决：** 使用 `ContentTemplate` + `DataTemplate` 包裹 `TextBlock`：
```xml
<ContentPresenter HorizontalAlignment="Center" VerticalAlignment="Center">
    <ContentPresenter.ContentTemplate>
        <DataTemplate>
            <TextBlock
                FontFamily="{Binding FontFamily, RelativeSource={RelativeSource AncestorType=Button}}"
                FontSize="{Binding FontSize, RelativeSource={RelativeSource AncestorType=Button}}"
                Foreground="{Binding Foreground, RelativeSource={RelativeSource AncestorType=Button}}"
                Text="{Binding}" />
        </DataTemplate>
    </ContentPresenter.ContentTemplate>
</ContentPresenter>
```

### 4. XamlParseException: 找不到资源

**原因：** 资源 Key 大小写不匹配（如 `DefaultFont` 已废弃、`iconfont` 应为 `IconFont`）。

**解决：** 所有字体 Key 严格使用 PascalCase，引用时大小写一致。
